MangaGuessr

Description du projet

MangaGuessr est une plateforme de jeu web interactive permettant aux fans de mangas de tester leurs connaissances sur leurs personnages préférés à travers différents modes de jeu captivants.

Contexte : Les fans de mangas recherchent des moyens ludiques de démontrer leur expertise sur leurs séries favorites. MangaGuessr répond à ce besoin en proposant une expérience de jeu engageante combinant reconnaissance visuelle et connaissance approfondie des univers manga (One Piece, Dragon Ball, Naruto).

Objectifs :

  • Créer une expérience de jeu immersive pour les passionnés de mangas.
  • Proposer plusieurs modes de jeu variés (classique, enchères, timeline).
  • Offrir une interface responsive et intuitive.
  • Gérer une base de données complète de personnages avec leurs attributs spécifiques.

Résultats obtenus : MangaGuessr propose un système de jeu complet avec comparaison chronologique des arcs, gestion des alias de personnages, et une interface React moderne capable de gérer plusieurs centaines de personnages avec des attributs variés.

Compétences acquises

  • Développement React/TypeScript : Architecture de composants réutilisables et typage strict pour la maintenabilité du code.
  • Gestion de données complexes : Structuration des ordres chronologiques d'arcs et mapping d'alias pour plusieurs univers.
  • Logique de jeu avancée : Implémentation d'algorithmes de comparaison pour les modes de jeu variés.
  • Design responsive : Interface adaptée à tous les appareils avec une excellente expérience utilisateur.
  • Optimisation des performances : Lazy loading, virtualisation de listes et debouncing pour une expérience fluide.
  • Gestion de projet Git : Workflow avec branches et intégration GitHub pour une collaboration efficace.

Défis techniques

1. Algorithme de comparaison chronologique multi-univers

Défi : Comparer chronologiquement des personnages issus de trois univers manga distincts (One Piece, Dragon Ball, Naruto), chacun ayant sa propre timeline avec des centaines d'arcs, sous-arcs et fillers.

Complexité technique :

  • Gestion d'un tableau unifié ARC_ORDER contenant 80+ arcs ordonnés chronologiquement.
  • Problème d'indexation : un personnage peut apparaître dans plusieurs arcs.
  • Ambiguïté des noms d'arcs (ex: "Pain" vs "Arc Pain", "Chūnin" vs "Chunin").

Solution implémentée :

  • Création d'un système de normalisation via ARC_ALIASES pour mapper les variantes vers un arc canonique.
  • Fonction compareArcs() avec logique ternaire retournant 'match', 'higher' ou 'lower'.
  • Fallback alphabétique pour les arcs non référencés.

2. Gestion de l'état asynchrone dans le mode enchères

Défi : Le premier personnage n'apparaissait pas lors de l'initialisation du mode enchères, causant un écran vide.

Complexité technique :

  • Race condition entre le chargement des données et le rendu React.
  • Le state initial null n'était pas correctement géré avant l'hydratation des données.
  • Problème de synchronisation entre useState et useEffect.

Solution implémentée :

  • Ajout d'un état de chargement intermédiaire pour gérer la transition.
  • Initialisation conditionnelle avec vérification de la disponibilité des données.
  • Utilisation de guards dans le rendu JSX pour éviter les accès à des propriétés undefined.

3. Système de typage strict pour les attributs de personnages

Défi : Chaque manga possède des attributs spécifiques (Devil Fruit pour One Piece, Chakra pour Naruto, Ki pour Dragon Ball) nécessitant un typage flexible mais sécurisé.

Complexité technique :

  • TypeScript nécessite des types explicites, mais les structures varient par série.
  • Risque d'erreurs runtime si un attribut manquant est accédé.
  • Difficulté à maintenir la cohérence lors de l'ajout de nouveaux personnages.

Solution implémentée :

  • Types discriminés avec union types pour chaque série.
  • Optional chaining (?.) systématique pour l'accès aux propriétés.
  • Validation des données à l'entrée avec type guards.

4. Optimisation du rendu pour les listes de personnages

Défi : Afficher et filtrer en temps réel des centaines de personnages sans latence perceptible.

Complexité technique :

  • Re-render complet à chaque frappe dans la recherche.
  • Images haute résolution causant des pics mémoire.
  • Scrolling saccadé sur mobile.

Solution implémentée :

  • Debouncing sur les inputs de recherche pour limiter les re-renders.
  • Lazy loading des images avec placeholder.
  • Virtualisation de la liste pour ne rendre que les éléments visibles.

Impact

Pour les joueurs :

  • Expérience ludique et éducative sur l'univers manga.
  • Différents modes adaptés à tous les niveaux de difficulté.
  • Interface fluide et réactive même sur mobile.

Pour la communauté manga :

  • Plateforme centralisée pour tester ses connaissances manga.
  • Engagement communautaire à travers le système de scores.

Points forts techniques :

  • Architecture scalable permettant l'ajout facile de nouveaux mangas.
  • Gestion efficace de données complexes sans compromis sur les performances.
  • Code TypeScript well-typed assurant la maintenabilité à long terme.
